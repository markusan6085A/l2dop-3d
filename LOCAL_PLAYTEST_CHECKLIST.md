# LOCAL PLAYTEST CHECKLIST (10 хв)

Швидкий ручний прогон перед merge або перед великим блоком фіч.

## 0) Базовий запуск

- `npm run dev`
- Перевірити, що `/health` відповідає `ok`.

## 1) Бій: rapid actions

- Відкрити бій з мобом у радіусі.
- Натиснути `attack/skill` 5-10 разів швидко.
- Очікування:
  - UI не зависає;
  - кнопки знову активні після відповіді;
  - немає “мертвого” стану після 409;
  - лог бою оновлюється коректно.

## 2) Екіп: rapid equip/unequip

- На сторінці персонажа швидко повторити `equip/unequip` 5-10 разів.
- Очікування:
  - немає залиплого lock;
  - стейт сумки/екіпу синхронний після 409;
  - `revision` у snapshot рухається вперед без дивних стрибків.

## 3) Телепорт + resync

- Зробити телепорт у місто.
- Повторити телепорт швидкими кліками.
- Очікування:
  - один успіх + контрольовані 409;
  - HUD і snapshot після телепорту актуальні;
  - після 409 дані самі ресинхроняться.

## 4) Сценарій refresh після 409

- Спровокувати 409 (rapid кліки), зробити refresh сторінки.
- Очікування:
  - персонаж підтягується без втрати критичного стану;
  - `expectedRevision` далі валідний;
  - можна продовжити гру без ручних “фіксів”.

## 5) Перевірка логів сервера

- Переглянути останні логи.
- Очікування:
  - є технічні поля `action/characterId/expectedRevision/actualRevision/result`;
  - немає `token`, `cookie`, повного `heroJson`, повного body.

## 6) Автоматичні smoke після ручного прогону

- `npm run build`
- `npm run test:integration:stability`
- `npm run test:stress:rapid`

## Definition of Done (локально)

- Ручний чек пройшов без UI-зависань.
- Smoke-команди зелені.
- Немає sensitive data в логах.
